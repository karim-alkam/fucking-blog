11:Te22,<p>If you have a headless Debian server (like a Raspberry Pi, a VPS, or an old laptop), eventually you’re going to want to see a desktop interface.</p>
<p>Most guides will tell you to install RealVNC or x11vnc. <strong>Don&#39;t do it.</strong></p>
<p>VNC transmits images of your screen. It is bandwidth-heavy, lags on WiFi, requires &quot;dummy plug&quot; drivers for headless setups, and breaks easily with modern display managers.</p>
<p>Instead, use <strong>XRDP</strong>. It uses the Microsoft Remote Desktop Protocol. It is faster, handles resolution scaling automatically, supports copy-paste out of the box, and doesn&#39;t require a physical monitor to be plugged in.</p>
<p>Here is how to set it up properly on Debian using <code>tasksel</code>.</p>
<h3 id="prerequisites">Prerequisites</h3>
<ul>
<li>A Debian server (CLI access via SSH).</li>
<li>A user account with <code>sudo</code> privileges.</li>
<li>A Windows PC (to connect from).</li>
</ul>
<hr>
<h3 id="step-1-install-the-desktop-environment-the-easy-way">Step 1: Install the Desktop Environment (The Easy Way)</h3>
<p>We will use <strong>Tasksel</strong>, a Debian tool that bundles packages into &quot;tasks.&quot; We are choosing <strong>XFCE</strong> because it is lightweight, stable, and works perfectly with remote connections.</p>
<ol>
<li><p>Update your repositories</p>
<pre><code>sudo apt update
</code></pre>
</li>
<li><p>Run Tasksel:</p>
<pre><code>sudo tasksel
</code></pre>
</li>
<li><p>A purple menu will appear. Use the <strong>Spacebar</strong> to select (add a <code>*</code> next to) these two items:</p>
<pre><code>[*] Debian desktop environment    
[*] Xfce
</code></pre>
<p> (Make sure GNOME is not selected to save resources).</p>
</li>
<li><p>Press <strong>Tab</strong> to highlight <code>&lt;Ok&gt;</code> and hit <strong>Enter</strong>. Go grab a coffee; this will download and install the GUI.</p>
</li>
</ol>
<hr>
<h3 id="step-2-install-the-rdp-server">Step 2: Install the RDP Server</h3>
<p>Now we install <strong>XRDP</strong> (the open-source RDP server) and the critical component most guides forget: <strong>dbus-x11</strong>.</p>
<p>Without <code>dbus-x11</code>, you will get a &quot;Unable to contact settings server&quot; error when you try to connect.</p>
<pre><code>sudo apt install xrdp dbus-x11
</code></pre>
<p>Enable and start the XRDP service:</p>
<pre><code>sudo systemctl enable xrdp
sudo systemctl start xrdp
</code></pre>
<hr>
<h3 id="step-3-configure-the-session">Step 3: Configure the Session</h3>
<p>By default, XRDP might not know which desktop environment to launch. We need to explicitly tell it to use XFCE for your user.</p>
<p>Run this command (as your normal user, <strong>not</strong> sudo):</p>
<pre><code>echo xfce4-session &gt; ~/.xsession
</code></pre>
<p>Now, restart the XRDP service to apply the changes:</p>
<pre><code>sudo systemctl restart xrdp
</code></pre>
<hr>
<h3 id="step-4-connect-from-windows">Step 4: Connect from Windows</h3>
<p>You don&#39;t need to download any third-party software.</p>
<ol>
<li>Open the <strong>Remote Desktop Connection</strong> app on Windows.</li>
<li>Enter the <strong>IP Address</strong> of your Debian server (e.g., <code>192.168.1.50</code>).</li>
<li>Click <strong>Connect</strong>.</li>
<li>Accept the security certificate warning.</li>
<li>When the XRDP login screen appears, enter your Linux username and password.</li>
</ol>
<h3 id="summary">Summary</h3>
<p>You now have a high-performance remote desktop. Unlike VNC, if you resize the window on your PC, the Debian desktop creates a new resolution to match it instantly. Enjoy!</p>
